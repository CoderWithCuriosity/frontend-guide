Learn how to use sessionStorage to store data for a single session. Data is cleared when the browser tab or window is closed.

Session Storage

sessionStorage works like localStorage but data is only available for the current session. When the user closes the browser tab or window, the data is cleared.

localStorage
Persists until manually cleared
localStorage.setItem("key", "value")
sessionStorage
Cleared when tab/window closes
sessionStorage.setItem("key", "value")

Console Output

// Session Storage Examples
> sessionStorage.setItem("temp", "123");
> let temp = sessionStorage.getItem("temp");
> console.log(temp);
"123"

> sessionStorage.removeItem("temp");
> console.log(sessionStorage.getItem("temp"));
null

// Comparing localStorage and sessionStorage
> localStorage.setItem("persist", "I persist");
> sessionStorage.setItem("temp", "I'm temporary");
> console.log(localStorage.getItem("persist"));
"I persist"
> console.log(sessionStorage.getItem("temp"));
"I'm temporary"

Key Takeaways

  • sessionStorage - Data is cleared when the tab/window closes
  • Same methods - setItem, getItem, removeItem, clear
  • Per-tab - Each tab has its own sessionStorage
  • JSON support - Use JSON.stringify() and JSON.parse() for objects
  • Security - Only accessible from the same origin
  • Use cases - Form data, temporary state, shopping cart
