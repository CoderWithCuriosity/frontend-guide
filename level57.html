Fetch API (POST)

POST requests are used to send data to a server to create new resources. You can send JSON data, form data, or other formats.

Try It Yourself

Syntax

// POST request with JSON data
fetch("https://api.example.com/posts", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    title: "New Post",
    body: "Content here"
  })
})
  .then(response => response.json())
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.log(error);
  });

Key Takeaways

  • method: "POST" - Specifies the request method
  • headers - Set Content-Type for JSON data
  • body - The data to send (must be stringified)
  • JSON.stringify() - Converts data to JSON string
  • response.json() - Parses the response
  • Check status - Server responds with status code

Your Challenge

Create a new file named level[level].html. Inside it, add a <script> tag and place your solution for this level within the script.

Practice POST requests by doing the following:

  • Send POST with Data: Create an object like let newPost = { title: "My New Post", body: "This is the content of my post", userId: 1 }. Send a POST request to https://jsonplaceholder.typicode.com/posts using fetch() with options: { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify(newPost) }. Log the response data to see what was created
  • Check Response Status: After sending your POST request, check the response status using response.status and response.ok. Log a message like "Status: " + response.status and "Success: " + response.ok. A status of 201 means "Created" which is typical for POST requests
  • Error Handling: Wrap your POST request in a try/catch block using async/await. Check if !response.ok and throw an error if the request fails. Catch any errors and log a user-friendly message like console.error("Failed to send data:", error.message)
  • Different Content-Types: Try sending data with different Content-Types. Send JSON data using "Content-Type": "application/json". Then try sending form data using let formData = new FormData(); formData.append("username", "john"); formData.append("email", "john@example.com") with "Content-Type": "application/x-www-form-urlencoded" or without setting Content-Type (browser will set it for FormData)
  • Chain Multiple Requests: First, send a POST request to create a new post. After getting the response, use the ID from the response (like data.id) to make a GET request for that specific post. Example: let postId = data.id; let getResponse = await fetch("https://jsonplaceholder.typicode.com/posts/" + postId). Then log the retrieved post to confirm it was created

💡 Hint: POST requests send data to create new resources on the server. The fetch options object needs method: "POST", headers to specify data format, and body with the data (as a string). Use JSON.stringify() for JSON data. Always check response.ok and handle errors. A successful POST usually returns a 201 status code. Chain requests by using the response data from the first request in the second one!
