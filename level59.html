Error Handling (try/catch)

Error handling allows you to catch and handle errors without crashing your program. Use try/catch to gracefully handle unexpected situations.

Try It Yourself

// Click a button to test error handling

Syntax

try {
  // Code that might throw an error
  let result = riskyOperation();
} catch (error) {
  // Code to handle the error
  console.log(error.message);
} finally {
  // Code that always runs
  console.log("Cleanup");
}

Console Output

// try/catch Examples
> try {
> let x = JSON.parse("invalid json");
> } catch (error) {
> console.log(error.message);
> }
Unexpected token i in JSON at position 0

// Custom error with throw
> try {
> if (age < 18) {
> throw new Error("Must be 18 or older");
> }
> } catch (error) {
> console.log(error.message);
> }
Must be 18 or older

// finally block
> try {
> console.log("Try block");
> } catch (error) {
> console.log(error);
> } finally {
> console.log("Always runs");
> }
Try block
Always runs

Key Takeaways

  • try - Wraps code that might throw an error
  • catch - Catches and handles errors
  • finally - Code that always runs (optional)
  • throw - Creates a custom error
  • Error object - Contains error details (.message, .name, .stack)
  • Graceful handling - Prevents program crashes
